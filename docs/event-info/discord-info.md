# Hack-Nation — 6th Global AI Hackathon (July 18–19, 2026)

## Overview
- Organizer: Hack-Nation, an initiative born out of the MIT Sloan AI Club.
- Format: a 24-hour hackathon sprint run simultaneously across ~12 physical hubs plus online participants (~2,000 hackers joined the kickoff call).
- Hub cities referenced in announcements: MIT, San Francisco, Stanford, New York City, London, Munich, Zurich/ETH, Paris, Dresden (plus others implied by the "12 hubs" branding — not all confirmed by name for this specific edition).
- Kickoff call took place July 18, ~17:00 ET.

## Registration & Platform
- Main platform: https://projects.hack-nation.ai/ — used for account login, challenge selection, API credits, and project submission.
- Each accepted participant receives a personalized login code by email (subject line: "Hack-Nation #6 starts tomorrow | Zoom Link & Details"). Check spam if it doesn't arrive.
- If you already registered via a Luma invite code, don't try to "reset password" — you must sign up fresh using the code; password reset emails won't be sent for accounts that were never fully created.
- No further registrations are being accepted once the event starts — only already-accepted participants can join a team.

## Teams
- Minimum team size: 1 (solo participation is allowed and welcomed).
- Maximum team size: 4.
- Solo builders are judged with the exact same criteria as full teams.
- Only team members listed on the final submission are eligible to receive prizes.
- Reusing your own previously written code is allowed, but the bulk of your build should be new work done during the hackathon — be ready to explain what was reused vs. newly built.

## The 6 Challenges
1. **Challenge 1 — ElevenLabs: "The Negotiator"**
   Help protect 28M Americans who move each year in a $20B+ market where comparable quotes vary by 5.6x and generate 13,000+ complaints annually. Build voice agents that call, compare, negotiate, and recommend the best evidence-backed deal.

2. **Challenge 2 — Maschmeyer Group: "The VC Brain"**
   Reimagine venture investing at 100x speed by enabling an evidence-backed decision on a $100,000 investment within 24 hours. Build an AI-first VC operating system for founder discovery, screening, diligence, and rapid investment decisions.

3. **Challenge 3 — RealPage: "RealDoor"**
   Prevent one documentation mistake from delaying an affordable-housing application for weeks, while keeping 100% of eligibility decisions with qualified humans. Build a renter-controlled copilot that explains requirements, checks documents, and prepares an application-ready packet.
   - A starter dataset/pack was shared in Discord: `RealDoor_Hackathon_Starter_Pack_v1.zip`.

4. **Challenge 4 — Databricks: "Data Legend"**
   Turn 10,000 healthcare-facility records into trusted planning and referral insights for a country of 1.4B people. Build a Databricks application that transforms fragmented data into evidence-backed, confidence-aware decisions.

5. **Challenge 5 — Hack-Nation: "Women's Hormonal Health"**
   Build research infrastructure for more than 50% of the global population and help unlock $1T+ in annual economic impact by closing the women's-health gap. Create an open dataset, benchmark, model, or application that advances women's hormonal-health research.

6. **Challenge 6 — Hack-Nation: "Genome Firewall"**
   Take on antibiotic resistance (4.7M+ deaths annually, 1M+ caused directly by resistance) and potentially deliver insights 1–3 days earlier. Build a defensive AI system that predicts whether antibiotics are likely to work or fail from a reconstructed bacterial genome.

Full challenge details/sign-up: https://projects.hack-nation.ai/#/companies (challenge briefs were also read out live on the kickoff call, not fully re-posted in text form in Discord).

## API Credits (OpenAI, ElevenLabs, Lovable, etc.)
- Go to projects.hack-nation.ai → "Submit Project" → **"My API Credits"** section.
- Fill out a short form (project vision, how you'll use the API, expected impact) for each API/tool you want credits for.
- After submitting the form you'll get a **redeem/promo code** and a "Redeem Credits" button/link.
- **OpenAI redemption steps** (per support): go to platform.openai.com/account/billing/overview → "Billing" → "Promo codes" / "Redeem code" (Promotions section) → paste the code → click Redeem. No card should be required once redeemed correctly.
- These OpenAI credits are for the **API platform** (building with GPT/multimodal models via API calls) — not for ChatGPT or Codex subscriptions.
- Known issue during the event: many participants got "invalid promo code" errors on both OpenAI and ElevenLabs. Support's fixes: clear your browser cache and retry, or wait for a new code to be issued. One moderator also suggested selecting the $25 OpenAI plan and applying the promo at checkout to bring the total to $0 — reported by some users to work, but treat as a community workaround rather than official instructions.
- A 429 "no quota" error when calling the OpenAI API generally means the credit code hasn't been successfully redeemed yet on that account.

### Submission checklist (format guide, still applicable)
| # | Item | Format | Purpose |
|---|---|---|---|
| 1 | Short description | Text, 150–300 words | High-level pitch for the jury |
| 2 | Demo video | Link, max 60 sec | Demonstrate your project |
| 3 | Tech video | Link, max 60 sec | Explain how you built it |
| 4 | 1-page report | PDF | Structured technical summary |
| 5 | GitHub repository | Public link | Source code and documentation |
| 6 | Zipped code | .zip file | Local backup of your project |
| 7 | Dataset (if used/generated) | Link or "N/A" | Raw or processed data |
| + | 1–3 slides + live demo | — | Shown during the live pitch session |

**Project Summary (150–300 words) should cover:** the problem you solved and its relevance, what you built during the hackathon, who benefits, and what works today. Tip: prioritize clarity over technical depth (save that for the tech video) — this is the first thing the jury sees.

**Demo video (≤60 sec):** ~5–10 sec problem intro, ~45–50 sec screen recording of the actual product/output, optional closing impact statement. Use Loom/OBS/Zoom; captions or voiceover help.

**Tech video (≤60 sec):** tech stack overview (10–15 sec), implementation highlights/what was hard (15–20 sec), challenges & limitations (15–20 sec), optional 5-sec reflection. Speak like a CTO giving a 1-minute investor update
